Posted on 2022-12-03 In Android Views:

这个文档的目的主要是为了记录为两个 Application 的 module 添加混淆支持的过程，同时也会添加混淆支持后遇到的 Crash 和解决方案，以便后续接手人员查缺补漏。

添加配置步骤

  • 基础参数设置
  • 通用混淆规则
  • Kotlin 混淆*
  • 第三方库或 SDK 混淆规则

这里的通用混淆规则网上一大堆，主要就是考虑以下几点

  • 系统组件等一些固定方法会被系统调用的
  • 反射和 Native 对应的类和方法
  • 自定义 View 等自定义类
  • 实体类与序列化的类

注:*是可选项，kotlin 携程一般不会报错，不过如果报 Module with the Main dispatcher is missing 就要把 kotlin 给 keep 一下
